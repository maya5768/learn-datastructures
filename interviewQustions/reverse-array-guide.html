איפה swap חייה בתוך reverse?

reverse מגדירה שני מצביעים ומריצה לולאה. בכל סיבוב היא קוראת ל-swap ומעבירה לה את המערך ואת שני האינדקסים. swap היא פונקציה נפרדת לחלוטין — היא לא יודעת על reverse, לא יודעת כמה גדול המערך, לא יודעת שאנחנו עושים היפוך. תפקידה יחידה: לקחת שני אינדקסים ולהחליף בין התאים.

שים לב לשמות הפרמטרים: reverse קוראת left ו-right — swap מקבלת אותם ומכנה אותם i ו-j. זה אותם ערכים, רק שמות שונים בתוך הפונקציה.

// reverse
int[] reverse(int[] arr) {
  int left = 0;
  int right = arr.length - 1;
  while (left < right) {
    swap(arr, left, right);  ← i=left, j=right
    left++;
    right--;
  }
  return arr;
}
// swap — void, לא מחזירה ערך
void swap(int[] arr, int i, int j) {
  int temp = arr[i];  ← שלב a
  arr[i] = arr[j];  ← שלב b
  arr[j] = temp;   ← שלב c
}

למה צריך 3 שורות ומשתנה temp? למה לא פשוט 2 שורות?

נניח שמישהו כתב ישירות, בלי temp:

// ❌ קוד שגוי

arr[j] = arr[i];  // arr[4]=arr[0]=5. BUG!

תוצאה: [5, 2, 3, 4, 5] — שני הקצות = 5. הערך 1 נעלם לתמיד.

אנלוגיה: שתי כוסות — א עם מיץ תפוז, ב עם מיץ לימון. רוצים להחליף. אם נשפוך א לתוך ב — איבדנו את הלימון. חייבים כוס שלישית ריקה (temp) לשמור אחד הנוזלים.

טבלת מעקב — כמה פעמים הלולאה רצה?

מערך [1, 2, 3, 4, 5] — גודל n=5. הלולאה תרץ ⌊5/2⌋ = 2 פעמים. בכל סיבוב reverse שולחת left ו-right ל-swap. swap מקבלת אותם כ-i וכ-j.

סיבוב left → i right → j arr[i] arr[j] temp מערך אחרי swap
1 0 4 1 5 1 [5, 2, 3, 4, 1]
2 1 3 2 4 2 [5, 4, 3, 2, 1]
עצירה 2 2 2 < 2? לא — הלולאה נגמרת

כלל כללי: עבור מערך בגודל n — הלולאה תרץ תמיד ⌊n/2⌋ פעמים. n=4 → 2. n=6 → 3. n=7 → 3 (אמצע לא מוחלף).
