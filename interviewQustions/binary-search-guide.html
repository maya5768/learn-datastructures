השווה arr[mid] למטרה

שלושה מקרים אפשריים:

arr[mid] == target → מצאנו! החזר mid
arr[mid] < target → חפש בימין: left = mid + 1
arr[mid] > target → חפש בשמאל: right = mid - 1

חזור עד ש-left > right

אם left עבר את right — המספר לא קיים במערך. מחזירים -1.

// מחזירה אינדקס של target, או -1 אם לא נמצא
1int binarySearch(int[] arr, int target) {
2int left = 0;
3int right = arr.length - 1;
4
5while (left <= right) {
6int mid = left + (right - left) / 2;

8if (arr[mid] == target) {
9return mid;
10} else if (arr[mid] < target) {
11left = mid + 1;
12} else {
13right = mid - 1;

15}

17return -1;

דרישה
מערך ממוין
למה left <= right ולא left < right?
כי כשנשאר אבר אחד — left == right — עדיין צריך לבדוק אותו!
למה mid = left + (right - left) / 2?
אם left ו-right גדולים מאוד, left + right גורם ל-Integer Overflow ב-Java.
סיבוכיות זמן — Binary Search

בכל צעד חוצים את מרחב החיפוש לשניים. לכן מספר הצעדים הוא log₂(n) — כמה פעמים צריך לחצות n כדי להגיע ל-1.

השוואה — Binary Search מול Linear Search

Binary Search

כללי Big O שחלים כאן
כלל 1
מורידים קבועים — O(2 log n) → O(log n)
כלל 2
חצייה בכל צעד = log — זה הסימן המזהה של Binary Search
כלל 3
O(log n) גדל לאט מאוד — מיליארד אברים זה רק ~30 צעדים!
