מה זו רשימה מקושרת?

רשימה מקושרת (Linked List) היא מבנה נתונים ליניארי שבו כל איבר (צומת) מצביע לאיבר הבא אחריו. בניגוד למערך, הנתונים אינם מאוחסנים בתאי זיכרון רציפים, אלא מפוזרים בזיכרון וקשורים זה לזה באמצעות מצביעים.

יתרונות: מאפשרת הוספה והסרה של איברים בקלות ובמהירות.
חסרונות: הגישה לאיבר מסוים יכולה להיות איטית יותר ממערך.

מבנה של צומת (Node)

כל צומת ברשימה מכיל שני חלקים:

  1. נתונים (Data) – הערך או המידע שנשמר בצומת.
  2. מצביע (Pointer) – כתובת הצומת הבאה ברשימה.

דוגמה לרשימה מקושרת:

7→ 5→ 8→ 3→ 2→ null

כל תיבה מכילה ערך + מצביע לצומת הבאה.

מימוש ב-Java

מחלקת IntNode:

public class IntNode {
    private int value;           // שדה המידע
    private IntNode next;       // שדה האדמיניסטרציה – מצביע לבא
    
    public IntNode(int val, IntNode n) {
        value = val;
        next = n;
    }
    
    public void setValue(int val) { value = val; }
    public void setNext(IntNode n) { next = n; }
    public int getValue() { return value; }
    public IntNode getNext() { return next; }
}

הסבר הקוד:

  • value – שומר את הערך (במקרה זה, מספר שלם).
  • next – מצביע לצומת הבא ברשימה.
  • הבנאי מקבל ערך ומצביע, ויוצר צומת חדש.
  • getters/setters – מאפשרים גישה מבוקרת לשדות הפרטיים.

יצירה והוספה של רשימה

הוספת איבר בראש הרשימה – זמן ריצה O(1):

public static void main() {
    IntNode list = null;                    // רשימה ריקה
    
    list = new IntNode(2, list);            // [2] → null
    list = new IntNode(3, list);            // [3] → [2] → null
    list = new IntNode(8, list);            // [8] → [3] → [2] → null
    list = new IntNode(5, list);            // [5] → [8] → [3] → [2] → null
    list = new IntNode(7, list);            // [7] → [5] → [8] → [3] → [2] → null
}

✅ הוספה בראש הרשימה: O(1) – פעולה קבועה ומהירה!

הוספת איבר בסוף הרשימה – זמן ריצה O(n):

IntNode p;
// עוברים עד לצומת האחרון
for(p = list; p.getNext() != null; p = p.getNext());

// יוצרים צומת חדש ומחברים אותו
IntNode temp = new IntNode(4, null);
p.setNext(temp);

⚠️ הוספה בסוף: O(n) – צריך לעבור על כל הרשימה כדי להגיע לסוף.

מחלקת IntList – עטיפה נוחה

במקום לעבוד ישירות עם IntNode, נכון יותר ליצור מחלקה IntList שמסתירה את מורכבות המבנה הפנימי. זה עוקב אחרי עקרון הכימוס (Encapsulation).

מחלקת IntList בסיסית:

public class IntList {
    private IntNode head;     // מצביע לראש הרשימה
    
    public IntList() {
        head = null;             // רשימה ריקה בהתחלה
    }
    
    // הוספת איבר לראש הרשימה
    public void addFirst(int n) {
        head = new IntNode(n, head);
    }
}

שימוש במחלקה:

public static void main() {
    IntList list = new IntList();
    list.addFirst(3);
    list.addFirst(7);
    list.addFirst(11);
    list.addFirst(24);
    list.addFirst(1);
    list.addFirst(7);
}
// תוצאה: [7] → [1] → [24] → [11] → [7] → [3] → null

פעולות נפוצות ברשימה מקושרת

1. ספירת איברים:

public int size() {
    IntNode p = head;
    int count = 0;
    while(p != null) {
        count++;
        p = p.getNext();
    }
    return count;
}

זמן ריצה: O(n)

2. הדפסת הרשימה:

public void print() {
    IntNode p;
    for(p = head; p != null; p = p.getNext())
        System.out.print(p.getValue() + "   ");
}

3. חיפוש מיקום של מספר:

public int num_position(int n) {
    IntNode p = head;
    int count = 1;
    
    while(p != null && p.getValue() != n) {
        p = p.getNext();
        count++;
    }
    
    if(p == null)
        return -1;      // לא נמצא
    return count;        // מיקום האיבר (מתחיל מ-1)
}

דוגמה: בהינתן הרשימה [7, 1, 24, 11, 7, 3], עבור 24 נחזיר 3, עבור 4 נחזיר -1.

מחיקת איבר מהרשימה

⚠️ הבעיה במחיקה פשוטה

אם פשוט נמחק את הצומת על ידי שינוי ה-next שלו ל-null, לא תהיה לנו גישה לשאר האובייקטים אחריו, וה"שרשרת" תיקרע!

✅ הפתרון: שני משתנים

  • משתנה p – רץ על הרשימה ומחפש את האיבר למחיקה.
  • משתנה prev – רץ תמיד צומת אחד לפני p.

כאשר נמצא האיבר, נעדכן את prev.next = p.next וכך "נדלג" על האיבר.

מימוש מחיקה:

public void remove(int n) {
    IntNode p, prev;
    p = head;
    prev = null;
    
    // חיפוש האיבר
    while(p != null && p.getValue() != n) {
        prev = p;
        p = p.getNext();
    }
    
    // אם האיבר לא נמצא
    if(p == null) {
        System.out.println("Node with value " + n + " not found.");
        return;
    }
    
    // אם זה האיבר הראשון
    if(prev == null) {
        head = p.getNext();
    } else {
        prev.setNext(p.getNext());
    }
}

הסבר הקוד:

  1. חיפוש: p זז קדימה עד שמוצא את הערך או מגיע לסוף. prev מתעדכן להיות תמיד לפני p.
  2. אם לא נמצא: מדפיסים הודעה ומחזירים.
  3. אם נמצא בראש (prev == null): מעדכנים את head להיות הצומת הבא.
  4. אם נמצא באמצע/בסוף: prev.next מצביע על הצומת שאחרי p.

מקרי קצה חשובים

🔑 בעת עבודה עם רשימות מקושרות, חשוב לטפל במקרים הבאים:

  • הרשימה ריקה (head == null)
  • טיפול באיבר הראשון (prev == null)
  • טיפול באיבר האחרון (p.next == null)

טבלת זמני ריצה

פעולה זמן ריצה הסבר
הוספה בראש O(1) רק יוצרים צומת חדש ומחברים
הוספה בסוף O(n) צריך להגיע לסוף הרשימה
חיפוש ערך O(n) במקרה הגרוע עוברים על כולה
מחיקת ערך O(n) צריך קודם לחפש
גישה לפי אינדקס O(n) אין גישה ישירה כמו במערך

השוואה: רשימה מקושרת vs מערך

פרמטר רשימה מקושרת מערך
גישה לאיבר לפי אינדקס O(n) O(1) ✅
הוספה בהתחלה O(1) ✅ O(n)
הוספה בסוף O(n) O(1) (אם יש מקום)
מחיקת איבר O(n) O(n)
גודל קבוע מראש? ❌ לא (דינמי) ✅ ✅ כן
זיכרון נוסף (pointers) צריך לא צריך ✅

💡 מתי להשתמש ברשימה מקושרת?

  • כשמספר האיברים משתנה הרבה (הוספה/הסרה תכופה).
  • כשרוב הפעולות הן בראש או בתחילת הרשימה.
  • כשאין צורך בגישה מהירה לפי אינדקס.
  • כבסיס למבני נתונים מתקדמים יותר (מחסנית, תור, עצים).
