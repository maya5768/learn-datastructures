🚶 תור (Queue)

FIFO – הכנס ראשון, הוצא ראשון

מה זה תור?

תור (Queue) הוא מבנה נתונים שבו ההכנסה מתבצעת בקצה אחד (הזנב) וההוצאה בקצה השני (הראש), בסדר FIFO (First In, First Out). כלומר, האיבר הראשון שנכנס הוא האיבר הראשון שיוצא.

התור מדמה תור אמיתי – כמו בסופרמרקט או בתחנה: הראשון בתור הוא הראשון שמקבל שירות.

הפעולות הבסיסיות

פעולה חתימה תיאור
enqueue void enqueue(int x) מוסיף את x לסוף התור (זנב)
dequeue int dequeue() מוציא ומחזיר את האיבר מראש התור
isEmpty boolean isEmpty() מחזיר true אם התור ריק

דוגמה להפעלת תור

נכניס לתור את האיברים 2, 5, 8, 4 (מהראשון לאחרון):

התור:

← 2 5 8 4 ←

ראש התור (יצא ראשון) ——————————— סוף התור (נכנס אחרון)

אם נבצע dequeue() – האיבר שיצא הוא 2 (הראשון שנכנס).

מימוש תור באמצעות רשימה מקושרת

נשתמש בשני מצביעים:

  • front – מצביע לראש התור (מקום היציאה).
  • back – מצביע לסוף התור (מקום הכניסה).

הדמיה חזותית:

התור ריק: front = back = null

אחרי enqueue(3):

front → 3 ← back

אחרי enqueue(6):

front → 3→6 ← back

אחרי enqueue(4):

front → 3→6→4 ← back

אחרי dequeue() (מחזיר 3):

front → 6→4 ← back

מימוש ב-Java:

public class Queue {
    private IntNode front;
    private IntNode back;
    
    // יצירת תור חדש
    public Queue() {
        front = back = null;
    }
    
    // בדיקה אם התור ריק
    public boolean isEmpty() {
        return front == null;
    }
    
    // הוספת איבר לסוף התור
    public void enqueue(int x) {
        IntNode temp = new IntNode(x, null);
        
        if(back == null) {
            // תור ריק – האיבר הוא גם הראש וגם הזנב
            front = back = temp;
        } else {
            back.setNext(temp);
            back = back.getNext();
        }
    }
    
    // הוצאת איבר מראש התור
    public int dequeue() {
        if(front == null)
            throw new RuntimeException("Underflow error");
        
        int x = front.getValue();
        front = front.getNext();
        
        // אם הוצאנו את האיבר האחרון, נאפס גם את back
        if(front == null)
            back = null;
        
        return x;
    }
}

הסבר הקוד:

  • enqueue: יוצרים צומת חדש. אם התור ריק, front ו-back שניהם מצביעים עליו. אחרת, מחברים לסוף הרשימה ומעדכנים את back.
  • dequeue: שומרים את הערך של front, מקדמים את front לצומת הבא. אם התור התרוקן – מאפסים גם את back.

מימוש תור באמצעות מערך

⚠️ הבעיה במימוש תמים

אם נשתמש במערך רגיל ונעדכן משתנה back בלבד, כל פעולת dequeue תחייב הזזה של כל האיברים תא אחד אחורה – פעולה של O(n)!

✅ הפתרון: מערך מעגלי

נשתמש במערך מעגלי: כאשר מגיעים לסוף המערך, חוזרים להתחלה. כך נמנעים מהזזת איברים.

מבנה המערך המעגלי:

נשתמש בארבעה משתנים:

  • front – אינדקס ראש התור.
  • back – אינדקס סוף התור (המקום הפנוי הבא).
  • capacity – גודל המערך הכולל.
  • size – כמה איברים יש בתור כרגע.

הדמיה:

אחרי 4 פעולות enqueue (5, 13, 24, 45):

[5, 13, 24, 45, _]
↑front         ↑back

אחרי enqueue(2) – back חוזר לתחילת המערך:

[5, 13, 24, 45, 2]
↑front&back

אחרי dequeue() – front מתקדם:

[_, 13, 24, 45, 2]
   ↑front      ↑back

מימוש ב-Java:

public class ArrayQueue {
    private int[] elements;
    private int front;
    private int back;
    private int size;
    private int capacity;
    
    // יצירת תור חדש בגודל N
    public ArrayQueue(int N) {
        elements = new int[N];
        front = back = 0;
        size = 0;
        capacity = N;
    }
    
    // הוספת איבר
    public void enqueue(int x) {
        if(size == capacity)
            throw new RuntimeException("Overflow error");
        
        elements[back] = x;
        back = (back + 1) % capacity;   // חישוב מעגלי!
        size++;
    }
    
    // הוצאת איבר
    public int dequeue() {
        if(size == 0)
            throw new RuntimeException("Underflow error");
        
        int x = elements[front];
        size--;
        front = (front + 1) % capacity;  // חישוב מעגלי!
        return x;
    }
    
    public boolean isEmpty() {
        return size == 0;
    }
}

הסבר הקוד:

  • מפתח האלגוריתם: (back + 1) % capacity – אם back מגיע לסוף המערך, מודולו מחזיר אותו ל-0.
  • size עוקב אחר מספר האיברים כדי לדעת אם המערך מלא או ריק.
  • בלי size, היה קשה להבחין בין מערך מלא לריק (בשניהם front == back).

תרגיל: מיון תור באמצעות תורים

שאלה: כתבו שיטה sort שמקבלת תור q וממיינת את איבריו בסדר עולה. מותר להשתמש בתורי עזר ובמשתנים, אסור להשתמש במערכים או מחרוזות. מותר מיון ב-O(n²).

פתרון:

public void sort(Queue q) {
    int min;
    Queue sorted = new Queue();
    Queue temp = new Queue();
    
    while(!q.isEmpty()) {
        min = q.dequeue();           // מתחילים מהאיבר הראשון
        
        // מוצאים את המינימום ב-q, שאר האיברים עוברים ל-temp
        while(!q.isEmpty()) {
            int x = q.dequeue();
            if(x < min) {
                temp.enqueue(min);
                min = x;
            } else {
                temp.enqueue(x);
            }
        }
        
        // המינימום עובר ל-sorted
        sorted.enqueue(min);
        
        // מחליפים בין q ל-temp
        Queue t = temp;
        temp = q;
        q = t;
    }
    
    // מחזירים את האיברים הממוינים חזרה ל-q
    while(!sorted.isEmpty())
        q.enqueue(sorted.dequeue());
}

הסבר הקוד:

  1. בכל איטרציה של הלולאה החיצונית, מוצאים את המינימום הנוכחי ב-q.
  2. שאר האיברים עוברים ל-temp.
  3. המינימום שנמצא מועבר ל-sorted.
  4. מחליפים בין q ל-temp ומתחילים איטרציה חדשה.
  5. בסוף, sorted מכיל את כל האיברים בסדר עולה – מעבירים אותם חזרה ל-q.

זמן ריצה: O(n²) (כמו מיון בחירה).

שימושים נפוצים בתור

  • תורי דפוס/הדפסה במערכות הפעלה.
  • BFS (חיפוש לרוחב) בגרפים ועצים.
  • תורי בקשות HTTP בשרתים.
  • תורי מוסיקה/וידאו – רשימות השמעה.
  • מערכות הזמנות בנקים, ממשלה, וכו'.
  • תורי הודעות במערכות מבוזרות (Message Queues).

השוואה: מחסנית vs תור

פרמטר מחסנית (Stack) תור (Queue)
סדר LIFO – אחרון נכנס, ראשון יוצא FIFO – ראשון נכנס, ראשון יוצא
מיקום הכנסה ראש המחסנית סוף התור
מיקום הוצאה ראש המחסנית ראש התור
