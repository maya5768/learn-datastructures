מה זה עץ חיפוש בינארי?

עץ חיפוש בינארי (BST - Binary Search Tree) הוא מבנה נתונים שבו ניתן לחפש, להוסיף ולהסיר ערכים בצורה יעילה. BST הוא סוג של עץ בינרי שבו לכל צומת יש לכל היותר שני "בנים".

מושגים בסיסיים בעצים

עץ לדוגמה:

       6   ← שורש (Root)
      / \
     4   9
    / \ / \
   3  5 7  8   ← עלים (Leaves)
   2

מושגים חשובים:

  • שורש (Root) – הצומת העליונה בעץ.
  • עלה (Leaf) – צומת ללא בנים.
  • גובה העץ – מספר הרמות בעץ (או מספר הקשתות מהשורש לעלה הרחוק ביותר).
  • אב (Parent) – צומת שיש לו בנים.
  • בן (Child) – צומת שיש לו אב.
  • תת-עץ (Subtree) – חלק של העץ שמתחיל מצומת מסוים.

מאפייני BST

🔑 שלושת הכללים של BST

  1. כל צומת יכול להכיל לכל היותר שני בנים (שמאלי וימני).
  2. תת-העץ השמאלי של כל צומת מכיל רק ערכים קטנים או שווים לערך הצומת.
  3. תת-העץ הימני של כל צומת מכיל רק ערכים גדולים או שווים לערך הצומת.

מחלקת Node

public class Node {
    private int value;
    private Node leftSon;
    private Node rightSon;
    
    public Node(int val) {
        value = val;
        leftSon = rightSon = null;
    }
    
    // Getters & Setters
    public int getValue() { return value; }
    public Node getLeftSon() { return leftSon; }
    public Node getRightSon() { return rightSon; }
    public void setLeftSon(Node n) { leftSon = n; }
    public void setRightSon(Node n) { rightSon = n; }
}

פעולות בסיסיות ב-BST

1. חיפוש (Search)

כדי למצוא ערך בעץ:

  1. מתחילים מהשורש.
  2. משווים את הערך המחפש לערך הצומת הנוכחי.
  3. אם הערך קטן יותר → ממשיכים לתת-העץ השמאלי.
  4. אם הערך גדול יותר → ממשיכים לתת-העץ הימני.
  5. אם שווה → מצאנו!

מימוש חיפוש ב-Java:

public boolean search(Node root, int x) {
    if(root == null)
        return false;             // לא נמצא
    
    if(root.getValue() == x)
        return true;              // נמצא!
    
    if(x < root.getValue())
        return search(root.getLeftSon(), x);  // חפש בשמאל
    
    return search(root.getRightSon(), x);     // חפש בימין
}

דוגמה לחיפוש הערך 91:

         19              91 > 19 → ימינה
        /  \
      16    81           91 > 81 → ימינה
     /  \   /  \
    9   18 90  98        91 < 98 → שמאלה
   /\         /
  2 12       91          91 = 91 → ✅ מצאנו!
      /
     7

2. הוספה (Insert)

ההוספה דומה לחיפוש – מחפשים את המקום הנכון ומוסיפים שם צומת חדש:

public void insert(int x) {
    root = insert(root, x);
}

public Node insert(Node r, int x) {
    if(r == null)
        return new Node(x);                         // מקום פנוי – יוצרים חדש
    
    if(r.getValue() > x)
        r.setLeftSon(insert(r.getLeftSon(), x));   // שמאלה
    
    if(r.getValue() < x)
        r.setRightSon(insert(r.getRightSon(), x)); // ימינה
    
    return r;
}

הסבר הקוד:

  • אם r == null – מצאנו מקום פנוי, יוצרים צומת חדש.
  • אם הערך קטן מהצומת – מבקשים רקורסיבית להוסיף בשמאל.
  • אם הערך גדול מהצומת – מבקשים רקורסיבית להוסיף בימין.
  • מחזירים את הצומת (עם השינויים) לקריאה הקודמת.

3. מציאת מינימום ומקסימום

ב-BST:

  • המינימום הוא הצומת השמאלי ביותר בעץ.
  • המקסימום הוא הצומת הימני ביותר בעץ.
public Node min(Node root) {
    if(root == null)
        return null;
    if(root.getLeftSon() == null)
        return root;
    return min(root.getLeftSon());
}

public Node max(Node root) {
    if(root == null)
        return null;
    if(root.getRightSon() == null)
        return root;
    return max(root.getRightSon());
}

מחיקה מ-BST (Delete)

המחיקה ב-BST היא הפעולה המורכבת ביותר. ישנם שלושה מקרים עיקריים:

מקרה 1: הצומת הוא עלה (אין לו בנים)

✅ פשוט מוחקים אותו ומעדכנים את הבן הרלוונטי של האב להיות null.

מקרה 2: לצומת יש בן אחד בלבד

💡 מחברים את ההורה של x ישירות לילד של x. כלומר, הילד של x תופס את מקומו.

מקרה 3: לצומת יש שני בנים

⚠️ המקרה המורכב ביותר. יש שתי דרכים לטפל:

  1. החלפה עם המינימום בתת-עץ הימני: מוצאים את הצומת השמאלי ביותר בתת-העץ הימני, מעבירים את ערכו לצומת המקורי, ומוחקים את הצומת שמצאנו.
  2. החלפה עם המקסימום בתת-עץ השמאלי: אותו רעיון, רק מהצד השני.

דוגמה: מחיקת 9 מעץ (שיש לו שני בנים)

שלב 1: מוצאים את המקסימום בתת-עץ השמאלי של 9 – זה 7.

שלב 2: מחליפים את הערך 9 בערך 7.

שלב 3: מוחקים את הצומת המקורי של 7 (שכעת הוא עלה).

כעת מבנה ה-BST נשמר.

סריקות עץ (Tree Traversals)

ישנן שלוש דרכים עיקריות לעבור על עץ, הנקראות מסלולים (Traversals):

  • Inorder: שמאל → שורש → ימין
  • Preorder: שורש → שמאל → ימין
  • Postorder: שמאל → ימין → שורש

Inorder – המסלול הכי חשוב!

✨ עובדה מרכזית: במסלול Inorder של BST, הערכים תמיד מוחזרים בסדר עולה!

אם קיבלנו רצף לא ממוין ב-Inorder – סימן שהעץ אינו BST.

מימוש Inorder ב-Java:

public void inorder(Node root) {
    if(root == null) return;
    
    inorder(root.getLeftSon());   // 1. סריקת תת-עץ שמאלי
    System.out.print(root.getValue() + " ");  // 2. שורש
    inorder(root.getRightSon());  // 3. סריקת תת-עץ ימני
}

דוגמה:

      4
     / \
    2   6
   /\   /\
  1 3  5  7

תוצאת Inorder: 1, 2, 3, 4, 5, 6, 7 ← סדר עולה ✅

Preorder

public void preorder(Node root) {
    if(root == null) return;
    
    System.out.print(root.getValue() + " ");  // 1. שורש
    preorder(root.getLeftSon());   // 2. שמאל
    preorder(root.getRightSon());  // 3. ימין
}

תוצאת Preorder על העץ לעיל: 4, 2, 1, 3, 6, 5, 7

Postorder

public void postorder(Node root) {
    if(root == null) return;
    
    postorder(root.getLeftSon());   // 1. שמאל
    postorder(root.getRightSon());  // 2. ימין
    System.out.print(root.getValue() + " ");  // 3. שורש
}

תוצאת Postorder על העץ לעיל: 1, 3, 2, 5, 7, 6, 4

ניתוח זמן ריצה

המקרה הטוב ביותר – עץ מאוזן

כאשר העץ מאוזן (כמעט "עץ בינארי מושלם"), גובהו הוא O(log n).

כל הפעולות מתבצעות בזמן O(log n).

עץ מאוזן (רמות לפי סדר הנדסי):

רמה 0: 1 צומת
רמה 1: 2 צמתים
רמה 2: 4 צמתים
רמה 3: 8 צמתים
...
סך הכל: 1 + 2 + 4 + 8 + ... = n
גובה: log₂(n)

המקרה הגרוע – עץ "מנוון"

אם מכניסים ערכים בסדר עולה או יורד, העץ נהיה כמו רשימה מקושרת!

זמן ריצה: O(n)

איזון עצים – עצי AVL (חומר מתקדם)

כדי לפתור את בעיית העץ המנוון, הומצאו עצים מאוזנים אוטומטית כמו AVL. עץ AVL הוא עץ חיפוש בינארי שבו לכל צומת, ההפרש בין גובה הבן השמאלי לבן הימני הוא 0, 1 או -1.

פקטור איזון: balance_factor = height(left) - height(right)

איך שומרים על איזון? – גלגולים (Rotations)

כאשר עץ מתחיל להיות לא מאוזן, מבצעים גלגול (Rotation). יש ארבעה מקרים עיקריים:

  1. LL: הנתיב שמאלה-שמאלה → גלגול ימינה.
  2. LR: הנתיב שמאלה-ימינה → גלגול שמאלה ואז ימינה.
  3. RR: הנתיב ימינה-ימינה → גלגול שמאלה.
  4. RL: הנתיב ימינה-שמאלה → גלגול ימינה ואז שמאלה.

מימוש גלגול שמאלה:

public void rotateLeft(Node p) {
    Node r = p.getRightSon();
    Node rl = r.getLeftSon();
    r.setLeftSon(p);
    p.setRightSon(rl);
}

💡 שימו לב: עצי AVL לא נכללים בחומר הבסיסי של הקורס, אך חשובים מאוד לראיונות עבודה.

תרגיל: בדיקה אם עץ הוא BST

שאלה: בהינתן עץ בינארי, איך בודקים ביעילות אם זהו עץ חיפוש בינארי?

פתרון באמצעות Inorder:

public boolean isBST(Node root) {
    ArrayList<Integer> list = new ArrayList<>();
    inorder(root, list);
    
    // בודקים אם הרשימה ממוינת בסדר עולה
    for(int i = 1; i < list.size(); i++) {
        if(list.get(i) < list.get(i - 1))
            return false;
    }
    return true;
}

private void inorder(Node root, ArrayList<Integer> list) {
    if(root == null) return;
    inorder(root.getLeftSon(), list);
    list.add(root.getValue());
    inorder(root.getRightSon(), list);
}

הסבר:

  • אוספים את כל ערכי העץ ב-Inorder לרשימה.
  • אם הרשימה ממוינת בסדר עולה → העץ הוא BST.
  • אחרת → העץ אינו BST.

💡 יתרונות וחסרונות של BST

יתרונות:

  • חיפוש יעיל ב-O(log n) בעץ מאוזן.
  • סדר טבעי של הנתונים (Inorder → ממוין).
  • הוספה ומחיקה דינמיות.

חסרונות:

  • הגובה יכול להגיע ל-n במקרה הגרוע.
  • דורש איזון תמידי לביצועים טובים (AVL, Red-Black).
