מהו הקורס?

הקורס עוסק בשני נושאים עיקריים:

  • אלגוריתמים – דרך יעילה לפתרון בעיה.
  • מבני נתונים – דרך יעילה לארגון הנתונים.

מהם מבני נתונים?

מבני נתונים הם דרכים לארגן ולאחסן מידע באופן שמאפשר לנו לגשת ולעבד אותו בצורה יעילה ומהימנה. כאשר אנחנו מדברים על "מבני נתונים", אנחנו מתכוונים לאופן שבו המידע מאורגן. ישנם סוגים רבים של מבני נתונים, כולל רשימות, טבלאות, עצים, ועוד.

למה מבני נתונים חשובים?

מבני נתונים חשובים מאוד בעולם המחשבים והתכנות מכיוון שהם מאפשרים לנו לנהל ולעבד מידע בצורה יעילה. באמצעותם ניתן:

  • לכתוב תוכניות מהירות ויעילות
  • לבנות אתרים ויישומים
  • לפתח פתרונות לבעיות מגוונות בתעשייה ובמחקר

סוגי מבני נתונים

שם המבנה תיאור
רשימות (Lists) אוסף של פריטים המאוחסנים בסדר מסוים. גמישות ונוחות לשימוש.
טבלאות (Tables) מבני נתונים בצורת שורות ועמודות. נפוצים בבסיסי נתונים.
עצים (Trees) מבנה נתונים היררכי עם צמתים וענפים.
מחסניות ותורים מבני נתונים ליניאריים עם חוקי גישה ייחודיים (LIFO/FIFO).
רשימות מקושרות מבני נתונים דינמיים שבהם כל איבר מצביע לאיבר הבא.

מהו אלגוריתם?

אלגוריתם הוא שיטה לפתרון בעיה באמצעות מחשב. בקורס נלמד:

  • שיטות לפיתוח ועיצוב אלגוריתמים
  • הוכחת נכונות של אלגוריתמים
  • חישוב זמן הריצה שלהם

דרכים לתיאור אלגוריתם

1. שפה טבעית (עברית/אנגלית וכו')

דוגמה: "נעבור על כל המספרים במערך ונסכם אותם אחד אחד."

יתרונות:

  • אפשר להביע הכל בשפה טבעית
  • אין צורך להיות מוגבלים לרישום טכני

חסרונות:

  • השפה נוטה להיות דו-משמעית ולא מדויקת
  • לעיתים התיאור כללי מדי ולא מספק

2. שפת תכנות (למשל Java)

דרך נוספת לתאר אלגוריתם היא לכתוב אותו בשפת תכנות. להלן האלגוריתם לסכימת איברי מערך ב-Java:

public static int sumArray(int[] a) {
    int sum = 0;
    for(int i = 0; i < a.length; i++) {
        sum += a[i];
    }
    return sum;
}

הסבר הקוד:

  • int sum = 0; – אתחול משתנה שישמור את הסכום.
  • for – לולאה העוברת על כל האיברים במערך.
  • sum += a[i]; – הוספת כל איבר לסכום הכולל.
  • return sum; – החזרת הסכום הסופי.

💡 נקודות חשובות

  • לכל מבנה נתונים יש פעולות עם זמני ריצה שונים.
  • בחירת מבנה הנתונים הנכון לבעיה היא קריטית ליעילות.
  • בקורס נלמד לבחור בין מבנים שונים לפי הדרישות.
