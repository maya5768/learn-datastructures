מה זה מיון מהיר?

מיון מהיר (Quick Sort) הוא אלגוריתם מיון יעיל המבוסס על גישת "הפרד ומשול". בניגוד למיון מיזוג, הוא ממיין במקום (in-place) ולא דורש זיכרון נוסף משמעותי.

רעיון האלגוריתם:

  1. בחירת ציר (Pivot): בחר איבר במערך שישמש כציר.
  2. חלוקה (Partition): העבר את כל האיברים הקטנים מהציר לתחילת המערך, ואת האיברים הגדולים ממנו לסוף.
  3. רקורסיה: קרא רקורסיבית לאלגוריתם על שני חלקי המערך (ללא הציר).

שלב ה-Partition (חלוקה)

מערך התחלתי: ננסה למיין את המערך סביב הציר (האיבר הראשון = 4):

4 10 15 2 7 8 14 3

אחרי partition (איברים < 4 משמאל, > 4 מימין):

2 3 4 15 10 8 14 7

הציר (4) מוצב במקומו הסופי! כעת ממיינים רקורסיבית את החלק השמאלי והימני.

מימוש ב-Java

שיטת quicksort הראשית:

public void quicksort(int[] a, int start, int end) {
    if(end <= start)
        return;                              // תנאי עצירה
    
    int p = partition(a, start, end);      // חלוקה וקבלת מיקום הציר
    
    quicksort(a, start, p - 1);           // מיון החלק השמאלי
    quicksort(a, p + 1, end);             // מיון החלק הימני
}

הסבר הקוד:

  • end <= start – תנאי העצירה: אם יש 0 או 1 איברים.
  • partition – מחלק את המערך ומחזיר את האינדקס שבו ממוקם הציר.
  • הקריאות הרקורסיביות ממיינות את שני החלקים (ללא הציר, כי הוא כבר במקומו).

שיטת ה-partition:

public int partition(int[] a, int left, int right) {
    int pivot = a[left];        // הציר – האיבר הראשון
    int p = left;
    int q = right;
    
    while(p < q) {
        // מחפש מימין איבר קטן או שווה לציר
        while(a[q] > pivot && p < q) q--;
        
        // מחפש משמאל איבר גדול מהציר
        while(a[p] <= pivot && p < q) p++;
        
        swap(a, p, q);           // מחליף ביניהם
    }
    
    swap(a, left, p);         // מציב את הציר במקומו
    return p;                  // מחזיר את מיקום הציר
}

הסבר הקוד:

  1. בחירת הציר: בחרנו את האיבר הראשון.
  2. שני מצביעים: p זז מהשמאל ימינה, q זז מהימין שמאלה.
  3. החלפה: כאשר p מוצא איבר גדול מהציר ו-q מוצא איבר קטן או שווה לציר, מחליפים ביניהם.
  4. סיום: כאשר p = q, מחליפים את הציר עם המיקום הזה.

פונקציית swap עזר:

public void swap(int[] a, int i, int j) {
    int temp = a[i];
    a[i] = a[j];
    a[j] = temp;
}

ניתוח זמן ריצה

מקרה ממוצע (Average Case):

כאשר הציר מחלק את המערך לחצאים שווים בערך:

T(n) = 2T(n/2) + n

לפי משפט האב (מקרה 2): T(n) = Θ(n log n)

מקרה גרוע (Worst Case):

כאשר הציר הוא תמיד האיבר המינימלי או המקסימלי (למשל, מערך כבר ממוין):

1 2 3 4 5 ... n

כל partition מחלק ל-0 ו-(n-1) איברים!

T(n) = T(n-1) + n = O(n²)

מקרה טוב (Best Case):

כאשר הציר הוא תמיד האיבר האמצעי בדיוק:

T(n) = 2T(n/2) + n = O(n log n)

התמודדות עם המקרה הגרוע – רנדומיזציה

⚠️ הבעיה

אם שיטת בחירת הציר צפויה (למשל תמיד האיבר הראשון), אפשר לבנות קלטים זדוניים שיגרמו למקרה הגרוע (O(n²)).

✅ הפתרון: ערבוב רנדומלי

נערבב את איברי המערך באופן אקראי לפני הקריאה הראשונית למיון המהיר. כך ההסתברות למקרה הגרוע נמוכה מאוד.

פונקציית shuffle:

public void shuffle(int[] a) {
    for(int i = 0; i < a.length; i++) {
        int j = random(0, a.length - 1);
        swap(a, i, j);
    }
}

הפונקציה עוברת על כל איבר ומחליפה אותו עם איבר אקראי אחר במערך, כך שמתקבל סידור רנדומלי.

בחירת ציר באמצע – דוגמה

אפשרות נוספת היא לבחור את האיבר האמצעי כציר:

int pivot = a[(left + right) / 2];

גם כאן יש מקרים גרועים! לדוגמה, בהינתן מערך של 10 איברים עם מבנה מסוים, בחירת הציר האמצעי תוביל לבחירת מקסימום או מינימום כל פעם.

טבלת זמני ריצה

מקרה זמן ריצה הסבר
מקרה טוב O(n log n) הציר מחלק תמיד לחצאים שווים
מקרה ממוצע O(n log n) הציר מחלק בערך לחצאים
מקרה גרוע O(n²) הציר הוא תמיד המינימום/מקסימום
זיכרון O(log n) עבור הקריאות הרקורסיביות

השוואה: Quick Sort vs Merge Sort

פרמטר Quick Sort Merge Sort
מקרה ממוצע O(n log n) O(n log n)
מקרה גרוע O(n²) O(n log n)
זיכרון נוסף O(log n) O(n)
In-place ✅ כן ❌ לא
Stable (יציב) ❌ לא ✅ כן
מהיר בפועל ✅ לרוב מהיר יותר ❌ לרוב איטי יותר

💡 מתי להשתמש בכל אחד?

  • Quick Sort: כשמהירות בפועל חשובה וזיכרון מוגבל.
  • Merge Sort: כשצריך מיון יציב או ביצועים עקביים במקרה הגרוע.
