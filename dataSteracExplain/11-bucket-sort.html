🪣 מיון דלי (Bucket Sort)

מה זה מיון דלי?

מיון דלי (Bucket Sort) הוא אלגוריתם מיון שעובד על ידי חלוקת האיברים ל"דליים" (Buckets) לפי ערכם, מיון כל דלי בנפרד, ואז שרשור הדליים הממוינים בסדר. האלגוריתם יעיל במיוחד כאשר הנתונים מתפלגים באופן אחיד (Uniform) בטווח ידוע.

בניגוד לאלגוריתמים כמו Merge Sort או Quick Sort שמסתמכים על השוואות, מיון דלי מסתמך על התפלגות הערכים, ולכן הוא יכול להגיע לזמן ריצה O(n) במקרים טובים.

שלבי האלגוריתם

📋 שלושה שלבים עיקריים

  1. יצירת דליים: יוצרים n דליים ריקים (רשימות מקושרות).
  2. פיזור לדליים: עוברים על המערך ומכניסים כל איבר לדלי המתאים לפי ערכו.
  3. מיון ואיחוד: ממיינים כל דלי בנפרד, ואז משרשרים את כל הדליים.

מימוש ב-Java

מחלקת DoubleNode (צומת לרשימה של double):

public class DoubleNode {
    private double value;
    private DoubleNode next;
    
    public DoubleNode(double val, DoubleNode n) {
        value = val;
        next = n;
    }
    
    public double getValue() { return value; }
    public DoubleNode getNext() { return next; }
    public void setNext(DoubleNode n) { next = n; }
}

מחלקת DoubleList (רשימה של double):

public class DoubleList {
    private DoubleNode head;
    
    public DoubleList() {
        head = null;
    }
    
    public DoubleNode getHead() {
        return head;
    }
    
    // הוספת ערך לרשימה במקום הנכון (מוין)
    public void addToList(double x) {
        if(head == null || head.getValue() >= x) {
            // הוספה בראש
            head = new DoubleNode(x, head);
            return;
        }
        
        // חיפוש המקום הנכון
        DoubleNode p = head;
        while(p.getNext() != null && p.getNext().getValue() < x)
            p = p.getNext();
        
        // הכנסה באמצע/בסוף
        p.setNext(new DoubleNode(x, p.getNext()));
    }
}

הסבר הקוד:

  • addToList – מוסיפה ערך לרשימה תוך שמירה על סדר עולה.
  • אם הרשימה ריקה או שהערך החדש קטן מהראש – מוסיפים בהתחלה.
  • אחרת, מחפשים את המקום הנכון ומכניסים שם.
  • יתרון: הרשימה תמיד ממוינת → אין צורך במיון נוסף!

מימוש Bucket Sort:

public static DoubleList[] bucketSort(double[] a) {
    int n = a.length;
    DoubleList[] buckets = new DoubleList[10];
    
    // שלב 1: יצירת הדליים
    for(int i = 0; i < 10; i++)
        buckets[i] = new DoubleList();
    
    // שלב 2: הכנסת כל איבר לדלי המתאים
    for(int i = 0; i < n; i++) {
        int index = (int)(a[i] * 10);  // חישוב אינדקס הדלי
        buckets[index].addToList(a[i]);
    }
    
    return buckets;
}

הסבר הקוד:

  1. יצירת 10 דליים: כל דלי הוא רשימה מקושרת ריקה.
  2. חישוב אינדקס הדלי: עבור ערך בטווח [0, 1), נכפול ב-10 ונעגל כלפי מטה.
    • 0.23 × 10 = 2.3 → אינדקס 2 (דלי 2)
    • 0.52 × 10 = 5.2 → אינדקס 5 (דלי 5)
  3. הכנסה ממוינת: שימוש ב-addToList שומר על הסדר בתוך כל דלי.
  4. בסוף, די לקרוא את הדליים בסדר 0→9 לקבלת המערך הממוין.

שרשור הדליים למערך סופי:

public static double[] collect(DoubleList[] buckets, int n) {
    double[] result = new double[n];
    int idx = 0;
    
    for(int i = 0; i < buckets.length; i++) {
        DoubleNode p = buckets[i].getHead();
        while(p != null) {
            result[idx++] = p.getValue();
            p = p.getNext();
        }
    }
    
    return result;
}

ניתוח זמן ריצה

מקרה ממוצע (התפלגות אחידה):

כאשר הנתונים מתפלגים באופן אחיד בטווח, צפוי שבכל דלי יהיו בערך n/k איברים (כאשר k הוא מספר הדליים).

  • זמן הפיזור: O(n)
  • זמן המיון של כל דלי: O((n/k)²) (מיון הכנסה בתוך רשימה)
  • סה"כ לכל הדליים: k · O((n/k)²) = O(n²/k)
  • זמן השרשור: O(n + k)

אם k = n, זמן הריצה הכולל: O(n)

מקרה גרוע:

כאשר כל האיברים נופלים לאותו דלי, מיון הדלי לוקח זמן של מיון רגיל: O(n²)

דוגמה: [0.91, 0.92, 0.93, 0.94, 0.95] – כולם נופלים לדלי 9.

טבלת זמני ריצה

מקרה זמן ריצה
מקרה טוב O(n + k)
מקרה ממוצע O(n + k)
מקרה גרוע O(n²)
זיכרון נוסף O(n + k)

כאשר n = מספר האיברים, k = מספר הדליים.

יתרונות וחסרונות

✅ יתרונות

  • מהיר במקרה ממוצע: O(n) כאשר הנתונים מתפלגים באופן אחיד.
  • פשוט להבנה וליישום יחסית לאלגוריתמים אחרים.
  • יציב (Stable) – שומר על הסדר היחסי של איברים שווים.
  • מתאים לנתונים מפולגים – טוב למשל למספרים ממשיים מפולגים אחיד.

⚠️ חסרונות

  • דורש הנחה על התפלגות הנתונים: אם הנתונים מקובצים, ביצועים פחות טובים.
  • זיכרון נוסף: דורש מקום לדליים.
  • לא מתאים לערכים שליליים או שטווח לא ידוע מראש (בלי התאמות).
  • תלוי בבחירת מספר הדליים: אם יש מעט מדי, ביצועים גרועים.

💡 מתי להשתמש במיון דלי?

  • כשהנתונים מתפלגים באופן אחיד בטווח ידוע.
  • כשצריך מיון מהיר ואין מגבלת זיכרון.
  • כשהנתונים הם מספרים ממשיים בטווח [0, 1) או דומה.
  • כשמהירות חשובה יותר מחיסכון בזיכרון.
