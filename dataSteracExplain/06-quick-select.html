מה זה Quick Select?

Quick Select הוא אלגוריתם לחיפוש האלמנט ה-k בגודלו מתוך רשימה לא ממוינת. האלגוריתם דומה במבנהו ל-Quick Sort, אך במקום למיין את כל הרשימה, הוא מתמקד רק במציאת האלמנט הרצוי.

🔑 הגדרת "האיבר ה-k הכי קטן"

בהינתן מערך המכיל n איברים שונים זה מזה, האיבר ה-k הכי קטן הוא איבר במערך ש-(k-1) איברים אחרים קטנים ממנו.

  • האיבר ה-1 הכי קטן = המינימום
  • האיבר ה-n הכי קטן = המקסימום

דוגמה

קלט:

7 4 8 21 3 1 9 15 12

quickSelect(list, 3) – נרצה את האיבר ה-3 הכי קטן

אם נמיין: [1, 3, 4, 7, 8, 9, 12, 15, 21] → התוצאה: 4

שלבי האלגוריתם

שלב 1: בחירת ציר (Pivot Selection)

בוחרים איבר מהמערך שישמש כציר. בחירה אקראית לרוב עובדת טוב.

שלב 2: חלוקה (Partition)

מחלקים את המערך לשני חלקים:

  • חלק אחד עם כל האלמנטים הקטנים מהציר.
  • חלק שני עם כל האלמנטים הגדולים מהציר.

שלב 3: חיפוש רקורסיבי בצד אחד בלבד

אחרי partition, הציר נמצא במקומו הסופי (במיקום curr). כעת:

  • אם curr - left == k - 1 – מצאנו את האיבר, החזר אותו.
  • אם curr - left > k - 1 – האיבר המבוקש בצד שמאל.
  • אם curr - left < k - 1 – האיבר המבוקש בצד ימין.

💡 בניגוד ל-Quick Sort שממשיך את שני הצדדים, Quick Select ממשיך רק בצד אחד!

מימוש ב-Java

שיטת kthSmallest:

public int kthSmallest(int[] a, int left, int right, int k) {
    int curr = partition(a, left, right);
    
    if(curr - left == k - 1)
        return a[curr];       // מצאנו!
    
    if(curr - left > k - 1)
        return kthSmallest(a, left, curr - 1, k);  // חפש בצד שמאל
    
    return kthSmallest(a, curr + 1, right, k - (curr - left + 1));  // חפש בצד ימין
}

הסבר הקוד:

  1. partition – מחלק את המערך ומחזיר את מיקום הציר.
  2. curr - left – מחשב כמה איברים יש משמאל לציר.
  3. אם כמות זו שווה ל-k-1, הציר הוא האיבר ה-k. מצאנו!
  4. אם יש יותר איברים משמאל מ-k, האיבר המבוקש נמצא בצד שמאל.
  5. אם יש פחות איברים משמאל מ-k, האיבר נמצא בצד ימין. חשוב: עדכן את k להיות k - (curr - left + 1) כי בצד ימין נפלנו על "מערך חדש".

פונקציית partition (זהה ל-Quick Sort):

public int partition(int[] a, int left, int right) {
    int pivot = a[left];
    int p = left;
    int q = right;
    
    while(p < q) {
        while(a[q] > pivot && p < q) q--;
        while(a[p] <= pivot && p < q) p++;
        swap(a, p, q);
    }
    
    swap(a, left, p);
    return p;
}

הדמיה חזותית של quickSelect(arr, 3)

מערך התחלתי: [7, 4, 8, 21, 3, 1, 9, 15, 12]

שלב 1: partition עם pivot=7

אחרי partition: [1, 4, 3, 7, 21, 8, 9, 15, 12]

הציר (7) במיקום 3, כלומר 3 איברים משמאלו.

בדיקה: curr - left = 3, k-1 = 2

3 > 2 → האיבר ה-3 הכי קטן נמצא בצד שמאל!

שלב 2: partition על [1, 4, 3] עם pivot=1

אחרי partition: [1, 4, 3]

curr - left = 0, k-1 = 2 → מחפשים בימין עם k חדש = 2

שלב 3: partition על [4, 3] עם pivot=4

אחרי partition: [3, 4]

curr - left = 1, k-1 = 1 → מצאנו! התשובה: 4

ניתוח זמן ריצה

מקרה ממוצע:

בממוצע, בכל שלב האלגוריתם מקטין את גודל המערך בחצי (הוא ממשיך רק בצד אחד):

T(n) = T(n/2) + n

לפי משפט האב (מקרה 3):

  • a = 1, b = 2, f(n) = n
  • log_b(a) = 0
  • f(n) = Ω(n^(0+1))
  • מקרה 3 → T(n) = Θ(n)

מקרה גרוע:

אם בכל פעם הציר הוא האיבר המינימלי/מקסימלי:

T(n) = T(n-1) + n = O(n²)

טבלת זמני ריצה

מקרה זמן ריצה
מקרה טוב O(n)
מקרה ממוצע O(n)
מקרה גרוע O(n²)

💡 יתרון מרכזי

במקום למיין את המערך כולו ב-O(n log n) רק כדי למצוא איבר אחד, Quick Select מוצא את האיבר ה-k הכי קטן ב-O(n) בממוצע – מהיר יותר!

שימושים נפוצים

  • מציאת המדיאן – האיבר ה-n/2 הכי קטן.
  • מציאת top-k elements – k האיברים הגדולים ביותר.
  • סטטיסטיקות – אחוזונים, רבעונים.
  • בחירת ציר טוב יותר ב-Quick Sort – בחירת המדיאן כציר.
