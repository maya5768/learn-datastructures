🔀 מיון מיזוג (Merge Sort)

אלגוריתם "הפרד ומשול" בזמן O(n log n)

מה זה מיון מיזוג?

מיון מיזוג הוא אלגוריתם מיון יעיל המבוסס על עיקרון "הפרד ומשול" (Divide and Conquer). האלגוריתם מחלק את המערך לשני חצאים, ממיין כל חצי בנפרד רקורסיבית, ואז ממזג את שני החצאים הממוינים לרצף ממוין אחד.

שלבי האלגוריתם

📋 שלושה שלבים

  1. Divide (חלוקה): חלק את המערך לשני חצאים באמצע.
  2. Conquer (פתרון): מיין רקורסיבית כל חצי בנפרד.
  3. Combine (מיזוג): מזג את שני החצאים הממוינים לרצף ממוין אחד.

הדמיה חזותית

מערך התחלתי:

1 7 4 9 3 5 8 2

↓ חלוקה לחצאים ↓

1 7 4 9       3 5 8 2

↓ חלוקה נוספת ↓

1 7    4 9       3 5    8 2

↓ חלוקה לאיברים בודדים ↓

1 7 4 9 3 5 8 2

↑ מיזוג וחזרה למעלה ↑

1,7    4,9       3,5    2,8

1,4,7,9       2,3,5,8

1 2 3 4 5 7 8 9

מימוש ב-Java

שיטת mergeSort הראשית:

public static void mergeSort(int[] a, int left, int right) {
    if(left < right) {
        int mid = (left + right) / 2;
        
        // מיון רקורסיבי של החצי השמאלי
        mergeSort(a, left, mid);
        
        // מיון רקורסיבי של החצי הימני
        mergeSort(a, mid + 1, right);
        
        // מיזוג שני החלקים הממוינים
        merge(a, left, mid, right);
    }
}

הסבר הקוד:

  • if(left < right) – תנאי העצירה: אם יש יותר מאיבר אחד להמיין.
  • int mid = (left + right) / 2; – חישוב נקודת האמצע.
  • שתי הקריאות הרקורסיביות ממיינות את שני החצאים.
  • merge – מבצע את המיזוג הסופי בין שני החצאים הממוינים.

שיטת המיזוג (merge):

public static void merge(int[] a, int left, int mid, int right) {
    int n1 = mid - left + 1;   // גודל החצי השמאלי
    int n2 = right - mid;        // גודל החצי הימני
    
    // יצירת מערכי עזר
    int[] leftArr = new int[n1];
    int[] rightArr = new int[n2];
    
    // העתקת הנתונים למערכי העזר
    for(int i = 0; i < n1; i++)
        leftArr[i] = a[left + i];
    for(int j = 0; j < n2; j++)
        rightArr[j] = a[mid + 1 + j];
    
    // מיזוג שני המערכים בחזרה למערך המקורי
    int i = 0, j = 0, k = left;
    while(i < n1 && j < n2) {
        if(leftArr[i] <= rightArr[j]) {
            a[k] = leftArr[i];
            i++;
        } else {
            a[k] = rightArr[j];
            j++;
        }
        k++;
    }
    
    // העתקת איברים שנותרו מהחצי השמאלי
    while(i < n1) {
        a[k] = leftArr[i];
        i++;
        k++;
    }
    
    // העתקת איברים שנותרו מהחצי הימני
    while(j < n2) {
        a[k] = rightArr[j];
        j++;
        k++;
    }
}

הסבר הקוד:

  1. יצירת מערכי עזר: מעתיקים את שני החצאים למערכים זמניים.
  2. מיזוג: עוברים על שני המערכים במקביל, ובכל פעם לוקחים את הקטן מביניהם.
  3. העתקת שאריות: אחרי שאחד המערכים הסתיים, מעתיקים את שארית המערך השני.

ניתוח זמן ריצה

נוסחת הנסיגה:

T(n) = 2T(n/2) + n
  • 2T(n/2) – שתי הקריאות הרקורסיביות על החצאים.
  • + n – הפעולה של merge שעוברת על כל n האיברים.

פתרון באמצעות עץ רקורסיה:

רמה 0: n (1 קריאה, גודל n)
רמה 1: n/2 + n/2 = n (2 קריאות, גודל n/2 כל אחת)
רמה 2: 4·(n/4) = n (4 קריאות, גודל n/4 כל אחת)
...
רמה k: 2^k · (n/2^k) = n
גובה העץ: log₂(n)

סכום כולל: n · log(n) = O(n log n)

פתרון באמצעות משפט האב:

  • a = 2, b = 2, f(n) = n
  • log_b(a) = log_2(2) = 1
  • f(n) = Θ(n^1) = Θ(n)
  • זה מקרה 2 של משפט האב
  • לכן: T(n) = Θ(n log n)

טבלת זמני ריצה

מקרה זמן ריצה
מקרה טוב (Best Case) O(n log n)
מקרה ממוצע (Average Case) O(n log n)
מקרה גרוע (Worst Case) O(n log n)
זיכרון נוסף O(n)

יתרונות וחסרונות

✅ יתרונות

  • זמן ריצה עקבי: תמיד O(n log n), גם במקרה הגרוע.
  • יציבות (Stable): שומר על הסדר היחסי של איברים שווים.
  • יעיל לרשימות מקושרות: אפשר למזג מבלי להעתיק.
  • מתאים לקבצים גדולים: ניתן להשתמש ב-External Merge Sort.

⚠️ חסרונות

  • זיכרון נוסף: דורש O(n) זיכרון נוסף למערכי העזר.
  • לא in-place: לא ממיין במקום (בניגוד למיון מהיר).

תרגיל: מיון מיזוג עם חלוקה לשלושה חלקים

אם נחלק את המערך לשלושה חלקים במקום שניים, נוסחת הנסיגה תהיה:

T(n) = 3T(n/3) + n

פתרון לפי משפט האב:

  • a = 3, b = 3, f(n) = n
  • log_b(a) = log_3(3) = 1
  • f(n) = Θ(n)
  • מקרה 2 → T(n) = Θ(n log n)

💡 מסקנה: חלוקה לשלושה חלקים לא משפרת את סדר הגודל האסימפטוטי – עדיין O(n log n).
